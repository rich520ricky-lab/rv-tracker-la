# LA RV Tracker - 設置指南\n\n## 概述\n\n這是一個自動化的 RV 車源追蹤儀表板，專門用於追蹤洛杉磯地區的 **2026 Winnebago Solis 59P** 車源。\n\n## 功能\n\n✅ **實時車源展示** - 展示來自 RVTrader 和授權經銷商的最新車源  \n✅ **智能排序** - 按價格或距離排序  \n✅ **收藏功能** - 保存感興趣的車源（LocalStorage）  \n✅ **自動更新** - 每天自動爬取最新數據  \n✅ **GitHub Pages 部署** - 免費託管，無需維護服務器  \n\n## 部署步驟\n\n### 1. 初始化 GitHub 倉庫\n\n```bash\ncd /home/ubuntu/rv-tracker-la\ngit init\ngit add .\ngit commit -m \"Initial commit: RV Tracker setup\"\ngit branch -M main\ngit remote add origin https://github.com/YOUR_USERNAME/rv-tracker-la.git\ngit push -u origin main\n```\n\n### 2. 啟用 GitHub Pages\n\n1. 進入 GitHub 倉庫設置\n2. 導航到 **Settings → Pages**\n3. 選擇 **Deploy from a branch**\n4. 選擇 **main** 分支和 **/ (root)** 目錄\n5. 點擊 **Save**\n\n### 3. 配置自動化工作流\n\n工作流文件已在 `.github/workflows/` 中創建：\n\n- **update-rv-data.yml** - 每天上午 9 點 UTC 執行爬蟲\n- **deploy-github-pages.yml** - 每次推送到 main 分支時部署\n\n### 4. 設置爬蟲腳本（可選）\n\n目前爬蟲使用示例數據。要集成真實數據源：\n\n```javascript\n// scripts/scrape-rv-data.js\n// 修改 scrapeRVData() 函數以調用 RVTrader API 或網頁爬蟲\n```\n\n## 訪問您的儀表板\n\n部署完成後，訪問：\n\n```\nhttps://YOUR_USERNAME.github.io/rv-tracker-la/\n```\n\n## 文件結構\n\n```\nrv-tracker-la/\n├── client/\n│   ├── src/\n│   │   ├── pages/\n│   │   │   └── Home.tsx          # 主儀表板頁面\n│   │   ├── components/           # UI 組件\n│   │   └── index.css             # 品牌色彩與排版\n│   └── public/\n│       └── data/\n│           └── listings.json     # 爬蟲生成的數據\n├── scripts/\n│   └── scrape-rv-data.js         # 數據爬蟲腳本\n├── .github/\n│   └── workflows/\n│       ├── update-rv-data.yml    # 每日更新工作流\n│       └── deploy-github-pages.yml # 部署工作流\n└── package.json\n```\n\n## 自定義\n\n### 修改更新時間\n\n編輯 `.github/workflows/update-rv-data.yml`：\n\n```yaml\non:\n  schedule:\n    -
